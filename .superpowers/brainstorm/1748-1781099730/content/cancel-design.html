Tombol Batalkan Pesanan hanya muncul saat status Menunggu Pembayaran. Klik → konfirmasi inline (tanpa modal). Konfirmasi → PATCH /api/orders/:id { status: "Dibatalkan" } → redirect ke menu (onMenu()).

Tombol Batalkan di header OrderDetail — tampil merah saat Menunggu Pembayaran, disabled + terkunci saat sudah dibayar/diproses/selesai. Owner menggunakan view Pesanan yang sama, sehingga perilakunya identik.

Perubahan teknis

  • ORDER_STATUS.CANCELLED = "Dibatalkan" ditambahkan ke types.ts
  • API PATCH /api/orders/[id] menolak cancellation jika status sudah bukan Menunggu Pembayaran (HTTP 409)
  • StatusPill mendapat case baru untuk Dibatalkan (warna merah/abu)
  • Filter Aktif di OrdersView mengecualikan Dibatalkan — order yang dibatalkan tidak muncul di list aktif
  • Customer: konfirmasi inline (no modal), lalu redirect ke menu via onMenu()
